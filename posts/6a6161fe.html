K8s集群搭建

k8s
CI CD容器构建
发布日期:   2022-11-06
更新日期:   2022-12-04
文章字数:   3.3k
阅读时长:   14 分
阅读次数:  

基于k8s搭建的一套CI/CD系统，其目的是方便k8s和服务端相关技术的实践，在搭建过程中会涉及docker、dockerhub、k8s、github、jenkins、kubesphere

一、介绍

k8s集群有两种方式

方式 优势 缺点
kubeadm 简单、快速 无法更好的理解k8s各个组件之间的关系
二进制包安装 繁琐、复杂 可能更好的理解k8s集群组件的关系

并且集群也分为简单版和复杂版，此教程采用kubeadm快速搭建一个简单集群（只有一个maste节点，两个工作节点）

1.硬件

注：机器至少是2核2G的配置

一台Mac物理机+3台Centos虚拟机，机器可以是虚拟机、三台物理机器或服务器。我使用的是三台centos7的虚拟机！

  • Mac物理机 安装了Docker、在Docker中启动Jenkins容器（使用windows、linux也是ok的）
  • 3台Centos虚拟机 部署k8s集群，集群中一个master节点，两个工作node节点
hostname ip地址 用途
server130 192.168.31.130 master节点
server131 192.168.31.131 node1节点
server132 192.168.31.132 node2节点

2. 软件

  • docker 容器引擎
  • dockerhub 官方的docker镜像注册服务器（可以在本地搭建Harbor代替）
  • k8s 一个容器编排管理工具，用于部署应用程序
  • github 源代码仓库（可以在本地搭建gitlab代替）
  • jenkins 发布工具
  • kubesphere k8s的web管理工具

二、k8s集群搭建

注意：除特别说明外，以下命令需在三台机器中都执行，建议找一个可以同时在多台机器中执行命令的工具（mac环境下可以使用item2支持多窗口同时输入命令）

# 打开这个功能的快捷键：会弹出告警信息，点OK确认。然后我们在第一个窗口中输入命令，会发现后面几个窗口都会自动同步输入命令。
⌘(command) + ⇧(shift) + i
# 关闭该功能的快捷键：

1. 安装docker

主要介绍，安装docker-ce社区版。

Docker 官方下载文档：https://docs.docker.com/get-docker/

docker-ce社区版安装步骤：https://developer.aliyun.com/mirror/docker-ce

  • [1] CentOS 7（使用 yum 进行安装）
# step 1: 安装必要的一些系统工具
sudo yum install -y yum-utils device-mapper-persistent-data lvm2
# Step 2: 添加软件源信息
sudo yum-config-manager --add-repo https://mirrors.aliyun.com/docker-ce/linux/centos/docker-ce.repo
# Step 3
sudo sed -i 's+download.docker.com+mirrors.aliyun.com/docker-ce+' /etc/yum.repos.d/docker-ce.repo
# Step 4: 更新并安装Docker-CE
sudo yum -y install docker-ce
# Step 4: 开启Docker服务
sudo service docker start
# 配置docker的阿里云镜像加速器地址
vi /etc/docker/daemon.json
#    内容如下
&#123;
  "exec-opts":["native.cgroupdriver=systemd"],
  "registry-mirrors": ["https://tn98v01s.mirror.aliyuncs.com"]
&#125;
# 重启docker
sudo systemctl daemon-reload
sudo systemctl restart docker
# 统一重启服务器
reboot

# 注意：
# 官方软件源默认启用了最新的软件，您可以通过编辑软件源的方式获取各个版本的软件包。例如官方并没有将测试版本的软件源置为可用，您可以通过以下方式开启。同理可以开启各种测试版本等。
# vim /etc/yum.repos.d/docker-ce.repo

2. 修改机器的网卡及host配置文件

  • [1] 如果安装后虚拟机ip与宿主机ip不一致，将网络类型该为“自动检测”即可
# 查看ip
ip addr
# 注意：如果出现虚拟机无法ping同主机的情况，说明主机与虚拟机不在一个网段上，将虚拟机的网段与主机的保持一致即可。比如主机为192.168.0.2，那么虚拟机就要为192.168.0.x

![image-20221203232216054](/Users/xzq/Library/Application Support/typora-user-images/image-20221203232216054.png)

![image-20221203232317617](/Users/xzq/Library/Application Support/typora-user-images/image-20221203232317617.png)

  • [2] 修改3台虚拟机的网卡配置文件
# 3台虚拟机分别在ifcfg-ens*配置文件中添加，之前安装都是ens33，这个版本是ens160。
vi /etc/sysconfig/network-scripts/ifcfg-ens160
# 添加如下内容
IPADDR=192.168.31.130 #静态IP，第一台ip：130，第二台ip：131，第三台ip：132
GATEWAY=192.168.31.1 #默认网关
NETMASK=255.255.255.0 #子网掩码 
# 重启网卡
service network restart
# 查询ip，发现配置的ip已经出现
ip addr
  • [3] 修改机器的hostname、hosts文件
# 使用vi分别编辑3台虚拟机的hostname文件
vi /etc/hostname
# 内容如下
server130 #第一台虚拟机
server131 #第二台虚拟机
server132 #第三台虚拟机
# 使用vi分别编辑3台虚拟机的hosts文件
vi /etc/hosts
# 添加如下内容
192.168.31.130 master
192.168.31.131 node1
192.168.31.132 node2
# 最后，执行ping master 看配置是否成功

3. 禁用防火墙、linux系统的安全服务、swap分区

  • [1]禁用防火墙
# docker会产生很多端口规则，为了引起不必要的麻烦，自己关闭掉（反正这个环境只是学习使用的，生成环境不要这么做）
systemctl stop firewalld
systemctl disable firewalld
systemctl stop iptables
systemctl disable iptables
  • [2]禁用linux系统的安全服务
# selinux是linux下的一个安全服务，安装软件会产生各种奇葩问题，直接禁用掉
vi /etc/selinux/config
# 设置如下内容
SELINUX=disabled
  • [3]禁用swap分区
# 在物理内存使用完之后虚拟磁盘空间作为内存使用，开启会产生性能影响，在搭建k8s集群的时候如果开启的话还需要特别说明
vi /etc/fstab
# 注释最后一行(/dev/mapper/cl_fedora-swap none                    swap    defaults        0 0)

# 执行如下命令，关闭swap
swapoff -a
# 重启网卡
service network restart

4. 修改Linux内核参数、配置ipvs功能

  • [1] 修改linux内核参数
# 修改linux内核参数，添加网桥过滤和地址转发功能
vi /etc/sysctl.d/kubernetes.conf
# 添加如下配置
net.bridge.bridge-nf-call-ip6tables = 1
net.bridge.bridge-nf-call-iptables = 1
net.ipv4.ip_forward = 1

# 重新加载配置
sysctl -p
#    加载网桥过滤模块
modprobe br_netfilter
# 查看网桥过滤模块是否加载成功
lsmod|grep br_netfilter

![image-20221204134144585](/Users/xzq/Library/Application Support/typora-user-images/image-20221204134144585.png)

  • [2] 配置ipvs功能
# 安装ipset、ipvsadm
yum -y install ipset ipvsadm
# 添加需要加载的模块，写入到脚本文件中
vi /etc/sysconfig/modules/ipvs.modules
# 内容如下
#!/bin/bash
modprobe -- ip_vs
modprobe -- ip_vs_rr
modprobe -- ip_vs_wrr
modprobe -- ip_vs_sh
modprobe -- nf_conntrack
# 注意：在网上找到教程安装K8S需要配置ipvs功能，但是在进行配置时会报错modprobe: FATAL: Module nf_conntrack_ipv4 not found。这是因为使用了高内核，较如博主就是使用了5.2的内核，一般教程都是3.2的内核。在高版本内核已经把nf_conntrack_ipv4替换为nf_conntrack了。

# 为脚本文件添加执行权限
chmod +x /etc/sysconfig/modules/ipvs.modules
# 执行脚本文件
/bin/bash /etc/sysconfig/modules/ipvs.modules
# 查询对应模块是否加载成功
lsmod |grep -e ip_vs -e nf_conntrack

5. 安装k8s组件

# 配置kubernetes的镜像源
cat > /etc/yum.repos.d/kubernetes.repo << EOF
[kubernetes]
name=Kubernetes
baseurl=https://mirrors.aliyun.com/kubernetes/yum/repos/kubernetes-el7-aarch64
enabled=1
gpgcheck=0
repo_gpgcheck=0
gpgkey=https://mirrors.aliyun.com/kubernetes/yum/doc/yum-key.gpg https://mirrors.aliyun.com/kubernetes/yum/doc/rpm-package-key.gpg
EOF

# 安装1.23.1指定版本的kubernetes相关组件
yum install -y kubelet-1.23.1-0 kubectl-1.23.1-0 kubeadm-1.23.1-0
# 安装最新版本的kubernetes相关组件
yum -y install kubeadm kubelet kubectl
# 配置kubelet的cgroup
vi /etc/sysconfig/kubelet
# 内容如下
KUBELET_CGROUP_ARGS="--cgroup-driver=systemd"
KUBE_PROXY_MODE="ipvs"

# 设置为开机自启动并启动 
systemctl enable kubelet

6. 集群初始化、安装网络组件

  • [1] 集群初始化（仅在master上执行）
# apiserver-advertise-address为master的ip地址
# 查看安装的k8s组件版本
kubectl version
# kubernetes-version为当前一步安装的k8s的版本,该步骤执行有点慢需要耐心等待一下
kubeadm init \
--apiserver-advertise-address=192.168.31.130 \
--image-repository registry.aliyuncs.com/google_containers \
--kubernetes-version v1.23.1 \
--service-cidr=10.1.0.0/16 \
--pod-network-cidr=10.244.0.0/16 \
--ignore-preflight-errors=all  \
--v=6
# 最后出现Your Kubernetes control-plane has initialized successfully!内容则表示集群初始化成功。最后按照安装成功后的提示语执行一些命令即可。若执行失败，需要重新初始化kubeadm reset # 先重置

![image-20221204162636498](/Users/xzq/Library/Application Support/typora-user-images/image-20221204162636498.png)

  • [2] 安装网络组件（仅在master上执行）
kubectl apply -f https://docs.projectcalico.org/manifests/calico.yaml
  • [3]将其他节点加入到集群中（在非master节点上执行）
# 这个命令在集群初始化成功后会提示 拷贝自己对应的命令执行即可
kubeadm join 192.168.31.130:6443 --token du6ab6.wbqkynebpm7268vt \

kubectl create deployment nginx --image=nginx
kubectl expose deployment nginx --port=80 --type=NodePort

二、基于k8s安装kubersphere

KubeSphere完全开源免费，是k8s的一个web管理界面，帮助企业快速构建一个强大和功能丰富的容器云平台。例如多云与多集群管理、Kubernetes 资源管理、DevOps、微服务治理、日志查询与收集、服务与网络、监控告警、存储管理、访问权限控制、GPU 支持、网络策略、镜像仓库管理以及安全管理等。

KubeSphere官方文档：https://kubesphere.com.cn/docs

1. Centos7搭建NFS服务端

注意：在k8s的所有节点上都需要安装nfs-utils、rpcbind

  • [1] 安装nfs-utils rpcbind
# 安装必要软件
yum -y install nfs-utils rpcbind
# 设置开机自启动
systemctl enable nfs
systemctl enable rpcbind

# 启动nfs
systemctl start rpcbind
systemctl start nfs
  • [2] 服务端配置nfs（仅在master上执行）
# 创建nfsdata服务端挂载目录
mkdir -p /usr/local/k8s/nfsdata
# 配置nfs
vi /etc/exports
/usr/local/k8s/nfsdata 192.168.31.0/24(rw,no_root_squash,no_all_squash,sync)
# 重启nfs
systemctl restart nfs

2. 创建基于NFS的DefaultStorageclass

kubesphere明确说明基于k8s安装需要配置DefaultStorageclass

  • [1] 下载资源文件
# 创建nfs-client目录
mkdir nfs-client
cd nfs-client
# 如果实在下载不下来，就直接进入github进行拷贝
for file in class.yaml deployment.yaml rbac.yaml test-claim.yaml test-pod.yaml; \
do wget https://github.com/kubernetes-retired/external-storage/tree/master/nfs-client/deploy/$file; \
done
  • [2] nfs-client目录下创建并修改deployment.yaml文件

![image-20221204172858756](/Users/xzq/Library/Application Support/typora-user-images/image-20221204172858756.png)

  • [3] nfs-client目录下创建rbac.yaml文件
apiVersion: v1
kind: ServiceAccount
metadata:
  name: nfs-client-provisioner
  # replace with namespace where provisioner is deployed
  namespace: default
---
kind: ClusterRole
apiVersion: rbac.authorization.k8s.io/v1
metadata:
  name: nfs-client-provisioner-runner
rules:
  - apiGroups: [""]
    resources: ["persistentvolumes"]
    verbs: ["get", "list", "watch", "create", "delete"]
  - apiGroups: [""]
    resources: ["persistentvolumeclaims"]
    verbs: ["get", "list", "watch", "update"]
  - apiGroups: ["storage.k8s.io"]
    resources: ["storageclasses"]
    verbs: ["get", "list", "watch"]
  - apiGroups: [""]
    resources: ["events"]
    verbs: ["create", "update", "patch"]
---
kind: ClusterRoleBinding
apiVersion: rbac.authorization.k8s.io/v1
metadata:
  name: run-nfs-client-provisioner
subjects:
  - kind: ServiceAccount
    name: nfs-client-provisioner
    # replace with namespace where provisioner is deployed
    namespace: default
roleRef:
  kind: ClusterRole
  name: nfs-client-provisioner-runner
  apiGroup: rbac.authorization.k8s.io
---
kind: Role
apiVersion: rbac.authorization.k8s.io/v1
metadata:
  name: leader-locking-nfs-client-provisioner
  # replace with namespace where provisioner is deployed
  namespace: default
rules:
  - apiGroups: [""]
    resources: ["endpoints"]
    verbs: ["get", "list", "watch", "create", "update", "patch"]
---
kind: RoleBinding
apiVersion: rbac.authorization.k8s.io/v1
metadata:
  name: leader-locking-nfs-client-provisioner
  # replace with namespace where provisioner is deployed
  namespace: default
subjects:
  - kind: ServiceAccount
    name: nfs-client-provisioner
    # replace with namespace where provisioner is deployed
    namespace: default
roleRef:
  kind: Role
  name: leader-locking-nfs-client-provisioner
  apiGroup: rbac.authorization.k8s.io
  • [4] nfs-client目录下创建class.yaml文件修改设置DefaultStorageClass
apiVersion: storage.k8s.io/v1
kind: StorageClass
metadata:
  annotations:
    storageclass.kubernetes.io/is-default-class: "true"   # 此属性为设置为default StorageClass
  name: managed-nfs-storage
provisioner: fuseim.pri/ifs # or choose another name, must match deployment's env PROVISIONER_NAME'
parameters:
  archiveOnDelete: "false"
  • [5] 部署deployment.yaml、rbac.yaml、class.yaml
# 部署命令
kubectl apply -f class.yaml
kubectl apply -f deployment.yaml
kubectl apply -f rbac.yaml
# 查看是否设置DefaultStorageClass成功
kubectl get sc
# 显示如下内容表示设置成功
NAME            PROVISIONER      RECLAIMPOLICY   VOLUMEBINDINGMODE   ALLOWVOLUMEEXPANSION   AGE
managed-nfs-storage (default)   fuseim.pri/ifs   Delete    Immediate        false    5m41s
